### Скрипты очистки баз данных и перепрошивки Accord

#### Список скриптов

Для функционирования всех скриптов необходимо наличие Аккорда в ЭВМ и подключенного к нему модуля accord-le.

* *acle-gx-erase-db.sh* - принудительно в заданном устройстве accord-le очищает базу данных для контролеров типа GX
* *acle-gxm2-erase-db.sh* - принудительно в заданном устройстве accord-le очищает базу данных для контролеров типа GXM2
* *acle-gx-update-firmware.sh* - скрипт обновления прошивки контроллеров типа GX
* *acle-gxm2-update-firmware.sh* - скрипт обновления прошивки контроллеров типа GXM2
* *acle-print-gx-list.sh* - печатает на экран список и название всех контроллеров
* *acle-functions.sh* - библиотека функций, которые используются в скриптах
* *amdz-gx-dump* - создает бинарник с дампом указанной зоны Аккорда

**acle-gx-erase-db.sh**
Скрипт очищает БД пользователей, БД контроля целостности и Журнал в Аккорде серии GX.
`sudo ./acle-gx-erase-db.sh [options]`
*options:*
- `-f` - принудительно очищать аккорд, даже если в ЭВМ вставлено несколько устройств(требуется подтверждение).
- `--force-local-utils` - принудительно использовать утилиты только из проекта

**acle-gxm2-erase-db.sh**
Скрипт очищает БД пользователей, БД контроля целостности и Журнал в Аккорде серии GXM2.
`sudo ./acle-gxm2-erase-db.sh [options]`
*options:*
- `-f` - принудительно очищать аккорд, даже если в ЭВМ вставлено несколько устройств(требуется подтверждение).
- `--force-local-utils` - принудительно использовать утилиты только из проекта

**acle-gx-update-firmware.sh**
Скрипт обновляет прошивку в Аккорде серии GX.
`sudo ./acle-gx-update-firmware.sh [options] image-file.bin`
*image-file.bin* - прошивка Accord'а серии GX, которую необходимо установить вместо текущей.
Внимание! При значительных отличиях старой и новой прошивок рекомендуется также очистить БД.
*options:*
- `-f` - принудительно прошивать аккорд, даже если в ЭВМ вставлено несколько устройств(требуется подтверждение)
- `--force-local-utils` - принудительно использовать утилиты только из проекта

*options:*

- `--force-local-utils` - принудительно использовать утилиты только из проекта

**acle-print-gx-list.sh**
Печатает на экран список и название все контроллеров серии Accord, установленных в ЭВМ.
Наличие драйвера accord-le необязательно для функционирования.
`./acle-print-gx-list.sh`

**acle-functions.sh**
Библиотека с функциями, которые используются в скриптах. В коде библиотеки оставлены комментарии
с краткими заметками об использовании данных функций при присоединении либы к скрипту.

**amdz-gx-dump**
Скрипт создает бинарник с дампом указанной зоны Аккорда.
`amdz-gx-dump.sh [flag] <altera|bios|log|user|kc|os|db|inaf_compt> <file.name> <gx|gxm2|auto> <input_db_file_path>`
- <altera|bios|log|user|kc|os|db|inaf_compt> - опция, конкретизирующая, дамп какой зоны необходимо сделать.
- <file.name> - имя файла, под которым будет сохранен дамп
- <gx|gxm2|auto> - [optional] используемый Аккорд. В режиме auto определяет серию автоматически (если не указан, используется auto).
- <input_db_file_path> - [optional] eсли указан gx или gxm2 аргумент, то можно четвертым аргументом передать путь до файла, который будет выбран источником данных. Будет использоваться указанная разметка
Также в режиме auto имеется возможность поставить флаг -f на месте [flag], чтобы была возможной работа, когда в ЭВМ находится несколько аккордов. Тогда дамп будет сделан у того, к которому подсоединен драйвер accord-le. В режиме auto всегда запрашивается подтверждение.
